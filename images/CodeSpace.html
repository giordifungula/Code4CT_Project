code4ct

If you have gone through all of the lessons up to this stage of the course then you should be a competent Web Developer. In fact, you should be better than most.

 

Brief:

 

The organisation Code4CT is rebuilding their website and they want it to look more modern and beautiful. Their idea is to have a more responsive, interactive and more engaging site, by having a website like that it will help users/visitors stay longer on the site and hence they will get the message and what Code4CT is for.

 

Project:

There is a design provided for you to work from.

Using what you have learned (HTML, CSS, CSS Grid, Command Line, Git, SASS, jQuery, and JS), create a website that looks exactly as the design. 

What's expected:

  • Create a new repository on github.com named code4ct
  • Create the appropriate files and folders you will need for this project. Well structured folders, files, and assets etc
  • Once you are done push your new changes to your repository. Remember to push constantly.
  • Make sure you commit and push your changes every time you make a new update to your site. 
  • Your website must be responsive, clear and neat CSS/SCSS selections, working across mobile, tablet, desktop,  
  • Homepage with a banner-image, counters, carousel (for the partners' logos) as seen on the mock-up
  • About page to include the team photos, the images should be optimized properly to fit without pixelation
  • Portfolio page - images to have a subtle zoom in effect on hover and when you click on them to open in pretty photo.
  • Contact Page - Must have a Google map showing the location of the organization, a contact form exactly as on the mock-up 
  • Button should expand when a user hovers over them from 100% to 105% to the right.
  • The navbar must be static or fixed and or it must be a sticky navbar such that when a user scrolls down the page the navbar follows their action.
  • Please add a logo on to the header
  • Back to Top button/icon should enable the user to go back to the top of a page and this action must be a smooth scroll.  
  • Courses Page:
    • Use object constructor to create a courses page,
    • User must be able to click on a register for course button,
    • Course will have a method which will add it to a 'basket' on the right-hand side.
    • A basket will show the number of courses signed up, duration, total cost.
    • If a course is signed up they can unsign from it as well...ie remove signup. that will reverse functionality by adding to basket.
    • Remember, objects are great programming tools. They can be used to encapsulate information about a particular thing without creating multiple variables.
    • Methods, Constructor functions, and prototypes might be the best way of making this page

 

Your facilitator will explain the project to you for better understanding before you start.

 

Make sure you chat about the decisions you made with your facilitator or code instructor around you if possible!

 

 

Once your code has been submitted it will be marked on the following criteria:

  • Functionality
  • Code Quality
  • Professional Conduct
  • Self-learning
